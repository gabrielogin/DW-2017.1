# DW-2017.1
Trabalho	de	DW	/	IC	/	UFFSistema	de	hospedagem	gratuita	ligadaa	esportesDescrição	do	sistema	
A)	Cadastro	dos	usuáriosOs	usuários	do	sistema	possuem	um	cadastro	com	suas	informações	básicas	(nome,	local	de	moradia,	fotos,	 etc.),	 bem	 como	 seu	 esporte	 favorito
Uma	 informação	 importante	 nos	 usuários	 é	 se	 estão	dispostos	a	receber	usuários	em	viagem	e	quantos.	
Existem	duas	formas	de	avaliar	usuários	(notas	de	0	a	 5	 estrelas):	 
  (i)	 usuários	 podem	 avaliar	 outros	 usuários	 simplesmente	 por	 conhecê-lo,	 escrevendo	algumtexto	a	respeito	deste	usuário;	
  (ii)	quando	um	usuário	fica	hospedado	na	casa	de	outro,	ao	final,	ambos	 emitem	 uma	 avaliação;	 
  (iii)	 quando	 há	 uma	 saída	 para	 a	 prática	 do	 esporte,	 ao	 final,	 ambos	emitem	uma	avaliação.	
  Nos	dois	últimos	casos,	os	usuários	somente	tem	acesso	a	avaliação	do	outro	após	emitir	a	sua,	e	a	avaliação	não	pode	sofrer	alteração.
  Desta	forma,	um	usuário	possui	5	tipos	de	média	de	avaliações	possíveis:	
  (i)	por	algum	amigo;	
  (ii)	quando	se	hospeda	na	casa	de	algum	usuário;	
  (iii)	quando	hospeda	algum	usuário;	
  (iv)	quando	participa	de	alguma	saída	para	prática	de	esporte;	e	
  (v)	quando	leva	usuários	para	a	prática	de	esporte.
B)	Solicitações	de	hospedagemA	 qualquer	 momento,	 um	 usuário	 viajante	 pode	 buscar	 um	 lugar	 para	 se	 hospedar,	 onde	 deve	 se	especificar	 o	 país	 e	 cidade	 de	 destino,	 procurando	 por	 usuários	 hospedeiros	 disponíveis	 para	hospedagem,	 e	 solicitando	 hospedagem	 para	 um	 período	 de	 datas	 específico.	 
Deve-se	 especificar	quantos	usuários	fazem	parte	da	viagem,	e	quantos	praticam	o	esporte	específico.Usuários	hospedeiros	devem	ter	uma	visualização	de	todos	as	solicitações	de	hospedagem	em	sua	casa,	contendo	as	informações	de	solicitação	descritas	acima,	podendo	aprovar	ou	reprovar	a	solicitação.Usuários	viajantes	devem	ter	uma	visualização	por	viagem	solicitada	(pode	haver	mais	de	uma)	de	todos	os	usuários	hospedeiros	solicitados	e	suas	ações	de	aprovação	ou	reprovação.	Este	pode	confirmar	uma	solicitação	 aprovada,	 medida	 que	 irá	 cancelar	 todas	 as	 outras	 solicitações	 ainda	 não	 avaliadas	 pelos	hospedeiros	(canceladas).Ao	 final	 do	 período	 de	 uma	 viagem,	 ambos	 usuários	 viajantes	 e	 hospedeiros	 podem	 se	 avaliar	mutuamente.C)	Saída	para	prática	de	esportesA	qualquer	momento,	um	usuário	pode	disparar	uma	convocação	de	saída	para	a	prática	de	esporte,	contendo	 o	 local	 da	 saída,	 a	 duração	 e	 o	 custo	 total	 da	 saída,	 incluindo	 rateio	 de	 aluguel	 de	 carro,	gasolina,	hospedagem,	etc.Ainda	vou	detalhar	mais	esta	parte	de	saída	para	prática	de	esporte.Objetivos	do	trabalhoDesenvolver	 o	sistema	 descrito	 acima	 utilizando	 o	 padrão	 arquitetural	 MVC	 (model-view-controller),	utilizando	HTML	forms	e	JSP	na	camada	de	interface	(view),	servlets	na	camada	de	controle	(controller)	e	classes	Java	na	camada	de	negócio	(model).	O	acesso	ao	banco	de	dados	pode	ser	feito	através	de	JDBC	ou	Hibernate	(JPA).Simplificações•O	esporte	será	fixo	o	surf,	não	há	necessidade	de	implementar	a	busca	por	esporte
•Uma	 saída	 para	 prática	 do	 esporte	 é	 uma	 simples	 descrição	 e	 haverá	 o	 preço	 total.	 Na	realidade,	tudo	que	há	custo	deveria	ter	uma	estimativa	e	o	valor	seria	dividido	igualmente	entre	os	participantes,	sendo	que	o	usuário	que	faz	a	proposta	pode	especificar	um	número	mínimo	e	máximo	de	participantes.ApresentaçãoPreparar	uma	rápida	apresentação	do	sistema(10	mins),	simulando	a	abrangência	dos	casos	de	uso	de	forma	a	demonstrar	as	capacidades	do	sistema	implementado.1o.
